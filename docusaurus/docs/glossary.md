# Robotics Glossary

A comprehensive glossary of 100+ terms used throughout the course.

## A

**Action** - A ROS 2 communication pattern for long-running tasks with feedback and cancellation capability.

**Actuator** - A mechanical component that creates motion (motors, servos, pneumatics).

**Autonomy** - Ability of a robot to operate without human intervention.

## B

**Bipedal** - Two-legged locomotion, as in humanoid robots.

## C

**Collision** - Detection of physical contact between objects in simulation or reality.

**Control Loop** - Repetitive process of sensing, computing, and actuating.

## D

**DDS (Data Distribution Service)** - Middleware protocol used by ROS 2 for communication.

**Digital Twin** - Virtual replica of a physical robot used for simulation and testing.

**DOF (Degrees of Freedom)** - Number of independent ways a robot can move.

## E

**Embodied AI** - AI systems that interact with the physical world through sensors and actuators.

**End Effector** - Tool or gripper at the end of a robot arm.

## F

**Forward Kinematics** - Computing end effector position from joint angles.

## G

**Gazebo** - Open-source robotics simulator with physics engine.

**Grasp Planning** - Computing how to grip an object securely.

## I

**IMU (Inertial Measurement Unit)** - Sensor measuring acceleration and rotation.

**Inverse Kinematics** - Computing joint angles to reach a desired end effector position.

**Isaac Sim** - NVIDIA's photorealistic robot simulator built on Omniverse.

## J

**Joint** - Connection between robot links allowing relative motion.

## K

**Kinematics** - Study of motion without considering forces.

## L

**LIDAR** - Laser-based distance sensor.

**Lifecycle Node** - ROS 2 node with managed states (configure, activate, deactivate).

## M

**Manipulation** - Robot tasks involving grasping and moving objects.

**Motion Planning** - Computing collision-free paths for robot movement.

## N

**Node** - Independent process in ROS 2 that performs computation.

## O

**Odometry** - Estimation of robot position from wheel encoders or sensors.

**Omniverse** - NVIDIA's platform for 3D simulation and collaboration.

## P

**Parameter** - Runtime configuration value for ROS 2 nodes.

**Perception** - Robot's ability to understand its environment through sensors.

**PhysX** - NVIDIA's physics simulation engine.

**Publisher** - ROS 2 node that sends messages on a topic.

## Q

**QoS (Quality of Service)** - Policies for reliable vs best-effort communication in ROS 2.

**Qdrant** - Vector database for storing embeddings.

## R

**RAG (Retrieval-Augmented Generation)** - AI technique combining search with language generation.

**ROS 2** - Robot Operating System 2, middleware framework for robotics.

## S

**SLAM (Simultaneous Localization and Mapping)** - Building a map while tracking robot position.

**Sensor Fusion** - Combining data from multiple sensors for robust perception.

**Service** -ROS 2 request-response communication pattern.

**Subscriber** - ROS 2 node that receives messages from a topic.

## T

**Topic** - Named channel for asynchronous message passing in ROS 2.

**Trajectory** - Planned path with time and velocity information.

## U

**URDF (Unified Robot Description Format)** - XML format for robot models.

## V

**VLA (Vision-Language-Action)** - AI models that map images and text to robot actions.

## X

**Xacro** - XML macro language for generating URDFs programmically.

## Z

**ZMP (Zero Moment Point)** - Point on ground where robot's weight appears concentrated, used for balance control.
